### ※  Java Web Application Architecture

---

![x](image2.png)

---

- 웹 브라우저가 용청하는 것은 웹 서버가 받는다. 웹 앱서버에는 응답하는 웹서버가 들어있고, Java App을 관리하는 관리자가 있다.

- 1. **웹 컨테이너 (Web Container)**
- **역할**: 웹 컨테이너는 주로 **서블릿(Servlet)**과 **JSP(JavaServer Pages)** 같은 웹 애플리케이션 컴포넌트를 실행하는 데 사용됩니다. 이 컴포넌트들은 웹 애플리케이션의 HTTP 요청을 처리하고, 동적인 콘텐츠를 생성합니다.
- **예시**:
    - **Tomcat**: Apache Tomcat은 가장 널리 사용되는 웹 컨테이너 중 하나로, 서블릿과 JSP를 실행할 수 있습니다.
    - **Resin**: Caucho에서 개발한 웹 컨테이너로, 서블릿과 JSP 실행을 지원합니다.
    - **Jetty**: 경량 웹 서버 및 서블릿 컨테이너로, 특히 임베디드 서버로 많이 사용됩니다.

- 2. **Java EE 서버 (Java EE Application Server)**
- **역할**: Java EE(Application Enterprise Edition) 서버는 웹 컨테이너의 기능뿐만 아니라 **EJB(Enterprise JavaBeans)**, **JMS(Java Message Service)**, **JTA(Java Transaction API)**, **JPA(Java Persistence API)** 등과 같은 추가적인 Java EE 사양을 지원합니다. Java EE 서버는 대규모 엔터프라이즈 애플리케이션을 위한 전체적인 플랫폼을 제공합니다.
- **예시**:
    - **WebLogic**: Oracle에서 제공하는 Java EE 애플리케이션 서버로, 엔터프라이즈급 애플리케이션을 실행할 수 있는 다양한 Java EE 기능을 제공합니다.
    - **WebSphere**: IBM에서 제공하는 Java EE 애플리케이션 서버로, 대규모 애플리케이션에서 요구되는 다양한 기능을 지원합니다.
    - **JBoss**: 현재는 WildFly로 알려진 이 오픈 소스 Java EE 애플리케이션 서버는 Red Hat에서 개발하며, Java EE 표준을 준수합니다.
    - **JEUS**: 대한민국의 TmaxSoft에서 개발한 Java EE 애플리케이션 서버로, 다양한 엔터프라이즈 애플리케이션을 지원합니다.
- **차이점 정리**
- **웹 컨테이너**: 서블릿, JSP 같은 **웹 컴포넌트**만을 실행하는 서버입니다. 주로 웹 애플리케이션에 집중되어 있으며, HTTP 요청 처리와 동적인 웹 페이지 생성이 주 역할입니다.
- **Java EE 서버**: **웹 컨테이너**의 기능을 포함하면서, 더 많은 엔터프라이즈 기능을 제공합니다. 데이터베이스 연동, 트랜잭션 관리, 메시징 서비스 등을 포괄하여 **대규모 애플리케이션**을 개발하고 실행할 수 있는 환경을 제공합니다.
- 결론
- **Tomcat**, **Resin**, **Jetty**는 주로 웹 애플리케이션만 실행하는 경량 서버(웹 컨테이너)입니다.
- **WebLogic**, **WebSphere**, **JBoss**, **JEUS**는 더 광범위한 기능을 제공하는 Java EE 애플리케이션 서버로, 대규모 엔터프라이즈 애플리케이션을 위한 다양한 기능을 포함하고 있습니다.

이 리스트에 있는 GlassFish, Geronimo, WebLogic, WebSphere, JBoss, JEUS는 모두 Java EE (현재 Jakarta EE) 애플리케이션 서버 또는 플랫폼입니다. 이들은 Java EE 사양을 구현하여 웹 애플리케이션, 엔터프라이즈 애플리케이션 및 서비스를 배포하고 관리하는 데 사용됩니다. 각 서버는 특정 기능, 성능, 지원 등을 제공하여 다양한 요구에 맞춰 사용할 수 있습니다. 아래에서 각 서버에 대해 간략히 설명하겠습니다:

1. **GlassFish**:
    - **설명**: 오픈 소스 Java EE 애플리케이션 서버로, Sun Microsystems(현재 Oracle)에 의해 개발되었으며, 현재는 Eclipse 재단의 관리 하에 있습니다.
    - **특징**: Java EE의 최신 사양을 빠르게 지원하며, 클라우드와 컨테이너 환경에서 잘 작동합니다.
2. **Geronimo**:
    - **설명**: Apache Software Foundation에서 관리하는 오픈 소스 Java EE 애플리케이션 서버입니다.
    - **특징**: 다양한 Java EE 기술을 지원하며, Apache 프로젝트의 일환으로 개발되었습니다.
3. **WebLogic**:
    - **설명**: Oracle에서 개발한 상용 Java EE 애플리케이션 서버입니다.
    - **특징**: 높은 성능, 안정성, 보안성을 제공하며, 대규모 엔터프라이즈 애플리케이션에 적합합니다.
4. **WebSphere**:
    - **설명**: IBM에서 개발한 상용 Java EE 애플리케이션 서버입니다.
    - **특징**: IBM의 다양한 소프트웨어와 하드웨어와의 통합이 뛰어나며, 엔터프라이즈 환경에서 널리 사용됩니다.
5. **JBoss** (현재 **WildFly**로 알려짐):
    - **설명**: Red Hat에서 개발한 오픈 소스 Java EE 애플리케이션 서버입니다.
    - **특징**: 상용 제품인 JBoss EAP(Enterprise Application Platform)과 오픈 소스인 WildFly가 있으며, 유연성과 확장성을 제공합니다.
6. **JEUS**:
    - **설명**: TmaxSoft에서 개발한 상용 Java EE 애플리케이션 서버입니다.
    - **특징**: 한국 및 아시아 시장에서 널리 사용되며, 높은 성능과 안정성을 제공합니다.

이 애플리케이션 서버들은 각각 고유의 기능과 장점을 가지고 있으며, 기업의 요구에 맞춰 적절한 서버를 선택하여 사용할 수 있습니다.

네, 맞습니다. Tomcat, Jetty, Resin, 그리고 Undertow는 모두 서블릿 컨테이너(Servlet Container) 또는 웹 서버(Web Container)로 분류됩니다. 이들은 웹 애플리케이션의 요청을 처리하고 서블릿을 실행하여 응답을 생성하는 역할을 합니다. 웹 컨테이너라는 용어는 일반적으로 서블릿 컨테이너와 동일하게 사용됩니다.
